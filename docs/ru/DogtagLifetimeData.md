# Данные жизненного цикла жетона (DogtagLifetime.json)

**Путь к файлу:** `$profile:EnhancedDogtag\Data\DogtagLifetime.json`

Файл данных времени выполнения, в котором сервер **автоматически сохраняет** время выживания, счёт убийств, уровень Death Wish и т.д. для каждого игрока. Это **данные времени выполнения**, а не файл конфигурации, поэтому ручное редактирование не рекомендуется.

---

## Назначение

- **Сохранение Death Wish:** При включённом `KeepDeathWishOnDeath` уровень Death Wish восстанавливается при следующем респавне после смерти
- **Сохранение счёта убийств:** При включённом `KeepKillCountsOnDeath` счёт убийств игроков/ИИ применяется к новому жетону при смерти
- **Сброс:** Данные жетонов всех игроков можно сбросить
- Применение предыдущих данных к жетону, создаётся при смерти игрока

---

## Структура JSON

```json
{
    "PlayerEntries": [
        {
            "PlayerName": "Ya-Mahaba",
            "PlayerId": "d2xG6LXwQPwK10j4pPC2LnItlC7RxZcmRCLyyqJaRNk=",
            "LifetimeSeconds": 3670,
            "TagType": "SMPZ_Dogtag_DeathWish_5",
            "BloodType": "None",
            "PlayerKillCount": 29,
            "AIKillCount": 819,
            "LastUpdateTime": "2026-3-5 14:39:50"
        }
    ]
}
```

---

## Описание опций PlayerEntries

| Опция | Тип | Описание |
|------|------|------|
| `PlayerName` | string | Имя игрока |
| `PlayerId` | string | Уникальный ID игрока (Base64) |
| `LifetimeSeconds` | int | Последнее время выживания (секунды) |
| `TagType` | string | Тип жетона (напр., `SMPZ_Dogtag_DeathWish_1` ~ `5`, базовый жетон) |
| `BloodType` | string | Группа крови (O+, O-, A+, A-, B+, B-, AB+, AB-, None) |
| `PlayerKillCount` | int | Счёт убийств игроков |
| `AIKillCount` | int | Счёт убийств ИИ |
| `LastUpdateTime` | string | Время последнего обновления |

---

## Примечания

- **Автогенерация:** Сервер автоматически обновляет при смерти игрока или создании жетона.
- **Макс. 1000 игроков:** Хранится до 1000 игроков. При заполнении удаляются старые записи по наименьшему времени выживания.
- **Ручное редактирование не рекомендуется:** Неверный формат или данные могут вызвать проблемы.
- **Config vs Data:** `Config\` — для настроек, `Data\` — для данных времени выполнения.

---

## Подробное поведение

**LifetimeSeconds:** Последнее время выживания игрока в секундах. Сохраняется время выживания на момент смерти.

**TagType:** Тип жетона для создания при следующем респавне. Хранится как `SMPZ_Dogtag_DeathWish_1` ~ `5` или базовый тип жетона.

**PlayerKillCount, AIKillCount:** Счёт убийств, применяемый к новому жетону при KeepKillCountsOnDeath = true.

**Интеграция KeepDeathWishOnDeath:** При true TagType, счёт убийств и т.д. в DogtagLifetime.json сохраняются после смерти и применяются к новому жетону при следующем респавне.
